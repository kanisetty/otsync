Working with help files

Welcome to the online documentation!

You can find the information you need quickly with the help of the following navigation elements:

The table of contents

You can define how detailed the table of contents (TOC) is displayed by selecting the Depth, on top of the TOC page:

  • shows only the main parts

  • includes the subordinate sections

  • shows all sections down to the third level

  • etc.

Simply click on the desired tab.

If there are more than five depth levels, the fifth tab shows all levels, so there are max. five depth tabs.

 

Примечания:

  • If you jump to a section that is deeper than the currently displayed table of contents, automatically the complete table of contents spanning all depth levels is opened. This also happens, if you click on a tab the depth of which is not sufficient for the currently displayed section.

  • Usually, the TOC highlights the page that is currently displayed, also after you browse or jump to another page. In some cases, e.g. after switching back from the Index page (see below) to a TOC page, this highlighting does not work. In this case, to re-sync the TOC, click once or twice on the lowermost entry in the navigation area of the help page.

 

The index

If you click on the tab next to the depth tabs, the Index is displayed instead of the table of contents (provided, the document has an index). You will get a list of important keywords together with links, so that you can navigate directly to the passage in the text where the word occurs.

The navigation area

Each help page contains two navigation areas: one at the top and one at the bottom. You can navigate to the next or previous section of the documentation by clicking on the arrows (upper right area) or links (lower area) for Next or Prev. In contrast to the symbols Back and Forward in the Internet browser, these links do not refer to the history of displayed pages, but rather browse through the documentation page by page as shown on the lowest level in the table of contents. That way you can browse through the sections like you would in a book.

Apart from the arrows for simple browsing, the upper navigation area provides additional means for navigating:

At the top, you find the title of the document/manual you are currently reading. Below the title, the individual hierarchical levels follow, down to the page currently displayed. So, below the document title, the subsection on the first level is displayed, followed by the subsection on the second level etc. If you click on such a title, you switch to the respective section.

To the left of the titles there are four small icons which allow you to browse – on the respective subsection level - to the preceding or subsequent section ( and ) or to move immediately to the table of contents of the section ( ). The icon corresponds with the section title itself, thus is a link to the beginning of the respective section.

If you move the mouse pointer over one of these icons without clicking on it, the title of the section the icon is linked to appears.

The bottom navigation area contains additional links, too: Home refers to the beginning of the documentation, whereas Up brings you to the next higher level in the table of contents. Simply click on the links to go to the desired page.

Navigation using the keyboard

In recent browser versions (IE6, Mozilla 1.7, Firefox), you can use the keyboard to browse through the document. There are two different browsing levels, the document level and the page level:

  • With Arrow up/down, you switch to the page level and scroll up or down on the currently displayed page.

  • With ESC, you switch back to the page level and then can use Page up/down or Arrow left/right to jump to the preceding or next page.

    On the document level, you also can use the number keys (1,2,3,...) to change to another depth in the table of contents, or the i key to display the index page.

Right after opening the document, you are on the document level and thus can browse through the pages using the page keys. But when you clicked on a page, you usually are on the page level; then first press the ESC key, if you want to browse through the pages of the document.

Referencing a specific page

In particular if you provide this HTML document on a central file share, you may want to refer to specific pages of this document from within your own documentation or in an E-Mail.

For this purpose, you only need to specify the URL of the respective page (which you easily can copy to the clipboard if you right-click on the lowermost title in the navigation area (see above) and choose “Copy Shortcut”, “Copy Link Location” or similar). Provided that Javascript is enabled in the Web Browser, the browser will automatically open the page then within the framework of the entire manual, i.e. also opens the table of contents on the left side.

 

Примечания:

  • If there is little space on your desktop, the navigation system allows you to easily navigate through the documentation even without the table of contents on the left side. Just move the separating line between the table of contents and the contents area as far to the left as possible, until the table of contents disappears.

  • The automatic functions of this online document are available only if JavaScript is active in your Web Browser. Even then, the one or other function possibly is not supported by some browser; this particularly applies to older browser versions (Netscape 6 and before, IE5 and before). The automatic functions include for example the highlighting the current section in the table of contents, the navigation using the keyboard as described above, and the automatic activation of the Table Of Contents in case that a specific page is directly opened in the browser.

    However, you need no JavaScript to read the document or to navigate through it. JavaScript is exclusively used to enhance the comfort.
