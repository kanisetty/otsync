i. 本書での表記法

[ユーザー インターフェイス]

この形式は、ボタン、アイコン名、メニュー項目、フィールドなどのグラフィカル ユーザー インターフェイス（GUI）に使用されます。

Filenames、commands、および sample data

この形式は、ファイル名、パス、URL、およびコマンド プロンプトのコマンドに使用されます。サンプル データ、テキスト ボックスに入力するテキスト、および他のリテラルにも使用されます。

 

注意:
PDF からコマンドライン サンプルをコピーする場合は、PDF には隠し文字が含まれていることがある点に注意してください。OpenText では、入手可能な場合は文書の HTML バージョンからコピーすることをお勧めします。

 
キー名

キー名は、すべて大文字で表されます。例えば、次のとおりです。
CTRL+V キーを押します。

<変数名>

角カッコ < > は、変数またはプレースホルダを示すために使用されます。ユーザは、カッコとその中の説明を適切な値で置き換えます。例えば、<server_name> は serv01 となります。

内部相互参照

相互参照をクリックすると、現在の文書の参照先に直接移動します。

外部相互参照

外部相互参照は、通常他の文書へのテキスト参照として使用されます。ただし、Knowledge Center などから文書が HTML 形式で入手可能な場合、外部参照は参照先文書の特定のセクションへのアクティブなリンクになっていることがあります。

警告、注意、ヒント
 

注意
警告は、元に戻せない問題を避けるのに役立ちます。この情報をしっかり読み、すべての指示に従ってください。

 
 

重要項目:
重要メモは、重大な問題を避けるのに役立ちます。

 
 

注意:
注意は、タスクに関する追加の情報です。

 
 

ヒント:
ヒントは、タスクをすばやく、または簡単に実行するための方法を示しています。
