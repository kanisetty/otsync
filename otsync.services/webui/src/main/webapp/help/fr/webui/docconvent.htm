Interface utilisateur

Ce format est utilisé pour des éléments de l'interface utilisateur graphique (GUI), tels que des boutons, des noms d'icône, des éléments du menu et des champs.

Filenames, commands et données d'exemple

Ce format est utilisé pour les noms de fichiers, chemins, URL et commandes de l'invite de commande. Il est également utilisé pour les données d'exemple, le texte à saisir dans des zones de texte et d'autres valeurs littérales.

 

Note :
Si vous copiez des exemples de lignes de commande depuis un document au format PDF, n'oubliez pas qu'un tel document peut contenir des caractères masqués. OpenText recommande de copier les lignes de commande depuis la version HTML du document, si possible.

 
NOMS DE TOUCHES

Les noms des touches apparaissent en MAJUSCULES, par exemple :
Appuyez sur CTRL+V.

<Nom de variable>

Les crochets < > sont utilisés pour indiquer une variable ou un espace réservé. L'utilisateur doit remplacer les crochets et le contenu descriptif par la valeur appropriée. Par exemple, <nom_serveur> devient serv01.

Lien hypertexte ou Weblink (http://www.opentext.com)

Ces formats sont utilisés pour les liens. Ces références actives permettent d'accéder à d'autres emplacements de la documentation (lien hypertexte) ou à d'autres ressources Internet (http://www.opentext.com).

Renvois internes

Cliquez sur le renvoi pour accéder directement à la référence du document actif.

Renvois externes

Les renvois externes sont des références à d'autres manuels généralement présentées sous forme de texte. Cependant, si un document est disponible au format HTML, dans le Knowledge Center par exemple, les références externes peuvent être des liens actifs permettant d'accéder à une section spécifique du document référencé.
