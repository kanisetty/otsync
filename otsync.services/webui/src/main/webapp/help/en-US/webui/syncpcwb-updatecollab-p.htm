Updating the Collaborators for a Shared Folder

To update the collaborators for a shared folder:

  1. Click a shared folder that you own.

  2. In the right pane, under COLLABORATORS, do one of the following:

    • To remove a collaborator, click next to the collaborator’s name and then click Unshare.

    • To add a collaborator, do the following:

      1. Enter a custom message when inviting new users to a share. If this field is left empty, no message will be included in the invitation email.

      2. Click and then click Read-Only to specify a read-only collaborator.

        When altering the permissions for a Content Server user in an Enterprise folder, if you grant the user read/write access when they currently have read-only access to the Enterprise folder, the additional permissions are applied. However, if you try to grant read-only access to a Content Server user who has permissions higher than See and See Contents, the change in permissions will not take effect. You need to alter the permissions at the folder location in the Enterprise Workspace in Content Server. For information on altering permissions in the Enterprise Workspace, see OpenText Content Server User Help - Using OpenText Tempo Box (SYNCLLES-H-UGD).

      3. In the box, type the name of the person with whom you want to share this folder. For an external user, type the user’s email address.

         

        Note:
        If the email address is ignored and auto complete fails to display the email address of the external user, you do not have permission to share folders with external users. Contact your administrator and request this privilege. For more information on external users, see Sharing with External Users.

         

      4. Click the user’s name or email address. A share invitation is sent. Collaborators must accept the share invitation before they can access the contents of the folder.

 

Note:
If there are more than seven collaborators for a folder, click Edit All [ ] Collaborators to update the collaborators.
